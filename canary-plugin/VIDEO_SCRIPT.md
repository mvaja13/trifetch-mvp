## VISUAL NOTES FOR RECORDING:

### Before Recording:
1. **Clear the data** - Delete `data/analyses.json` for a fresh demo
2. **Prepare audio file** - Have a sample audio file ready
3. **Test the flow** - Do a practice run

### Camera Work:
- **Use smooth screen recording** (QuickTime/OBS at 60fps)
- **Cursor movements should be deliberate** and smooth
- **Zoom in on key features** when highlighting them
- **Use transitions** between major sections (fade/cut)

### Key Screens to Capture:
1. Homepage with upload area
2. Real-time progress bar in action
3. Complete analysis results (scroll smoothly)
4. Interactive timeline chart with data
5. Audio waveform player
6. Analysis history with multiple entries
7. PDF export button click

### Audio Recording Tips:
- **Speak with confidence and energy**
- **Pace: Slightly faster than normal** (creates excitement)
- **Emphasize key phrases**:
  - "48 hours"
  - "Production-ready"
  - "Real-time"
  - "Your API can power"
- **Professional tone** but not stiff
- **End with enthusiasm** on the call to action

## PRO TIPS FOR MAXIMUM IMPACT:

### 1. **Start STRONG**
- First 3 seconds determine if they keep watching
- Lead with the "48 hours" hook - it's impressive

### 2. **Show, Don't Tell**
- Let the UI speak for itself
- Smooth interactions > long explanations

### 3. **Emphasize Integration**
- Keep saying "your API" to make it personal
- Show how seamlessly it integrates

### 4. **End with Action**
- Make them want to reply
- Create curiosity about "what else is possible"

### 5. **Technical Credibility**
- Mention WebSocket, TypeScript, production-ready
- Shows you're serious, not just making a mockup
